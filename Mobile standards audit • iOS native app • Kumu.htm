        Issues

        Issues allow you (and anyone with access to the project) to raise concerns, challenge assumptions, and engage multiple perspectives. Think of them as the fuel driving changes to your map when you can't all be in one room.

        We'll soon have issues tied directly into the app, but until then we recommend opening them in a new tab so it's easy to jump back and forth between the issues and the maps.

        View issues in new tab

        Wait a sec... where did discussions go?

        If you're a long time Kumu user you know we originally called these things discussions. Discussions are important but making sure discussions are actionable and stay focused is even more so. We think "issues" captures that spirit better than "discussions" do.

        Plus you always felt like an a-hole closing a discussion. Closing an issue doesn't mean the discussion is over—it just means the issue has been resolved, for now.

        Fork this project

        A fork is a copy of a project. Forking a project allows you to experiment with changes without affecting the original project. Forks are commonly used to propose changes to someone else's project or to use someone else's project as a starting point for your own.

          Help

          We strive to keep Kumu as easy to use as possible but we know there's a lot to digest. The quickest way to get up to speed is to watch the short intro video below and then play around in a dummy project (public projects are free and can easily be deleted when you're done). If you get stuck the help guides at docs.kumu.io will get you moving again. We've also included a list of handy keyboard shortcuts below. If you still have any questions just ask!
